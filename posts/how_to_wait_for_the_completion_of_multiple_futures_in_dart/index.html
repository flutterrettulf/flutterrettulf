Asynchronous Programming in Dart

Asynchronous programming is a method of writing code, that allows the program to run multiple tasks in the background without blocking the main thread. This means that the program can continue to respond to user requests, while it is waiting for the completion of a long running task.

For example, consider a mobile application that downloads a file from the internet. In a synchronous program, the program will wait for the file to download before it can respond to user requests. That is, the program will be blocked until the file is downloaded. It will not be able to respond to user requests until the file is downloaded. This is not a good user experience.

With asynchronous programming, the program can continue to respond to user requests, while it is downloading the file. This is a better user experience.

In Dart, asynchronous programming is achieved using the Future class. A Future object represents a value that will be available in the future. The Future class is defined in the dart:async library. You can import the library using the following statement:

import 'dart:async';

Future<int> getFutureValue() async {
    // network requests
    // file operations
    // database operations
}

//usage

s =  await getFutureValue();

// or

getFutureValue().then((value) {
    s = value;
});

Future.wait() method

Sometimes, our program has to make multiple network requests. We have to wait for the completion of all the network requests before we can proceed.

For example, consider a mobile application that displays a list of courses. The application has to make a network request to fetch the list of courses. Simultaneously, the application might have to make a network request to fetch the list of instructors. Suppose, we want to display the list of courses and the list of instructors once both the network requests are completed.

With the Future.wait() method, we can wait for the program to complete both the network requests. The method takes a list of Future objects as an argument. It returns a Future object that completes when all the Future objects in the list are completed. When the Future object completes, it returns a list of values. The values are returned in the same order as the Future objects in the list.

import 'dart:async';

Future<int> getFutureValue1() async {
    // some asynchroneous code
}

Future<int> getFutureValue2() async {
    // some asynchroneous code
}

Future<int> getFutureValue3() async {
    // some asynchroneous code
}

//usage

Future.wait([
        getFutureValue1(), 
        getFutureValue2(), 
        getFutureValue3()
    ]).then((values) {
    print(values);
});

// or 

values = await Future.wait([
        getFutureValue1(), 
        getFutureValue2(), 
        getFutureValue3()
    ]);

In the above example, we created three functions that performs some kind of asynchronous operation. All the functions return a Future object. We passed the Future objects as a list to the Future.wait() method. The method returns a Future object that completes when all the three asynchronous operations are completed. When the Future object completes, it returns a list of values. The values are returned in the same order as the Future objects in the list.

Summary

In this tutorial, we learned how to wait for the completion of multiple futures in dart/Flutter. We used the Future.wait() method to wait for the completion of all the Future objects in a list. The method returns a Future object that completes when all the Future objects in the list are completed. When the Future object returned by the Future.wait() method completes, it returns a list of values. The values are returned in the same order as the Future objects in the list.
