Creating an App

After installing flutter, you will be able to start making apps with Flutter. We will learn how to create a basic app with this tutorial. The app will have a simple button which will respond on click with the message “Hello Flutter”.

From the Terminal

Flutter provides a command-line tool that lets you start and manage your app. After installing Flutter you can start making apps with the following command.

flutter create hello_world
cd hello_world

From VS Code​

If you have setup VS code with Flutter then you can create a flutter app from it as well.

  • Invoke View > Command Palette.
  • Type “flutter”, and select the Flutter: New Project.
  • Select Application.
  • Create or select the parent directory for the new project folder.
  • Enter a project name, such as my_app, and press Enter.
  • Wait for project creation to complete and the main.dart file to appear.

You can double click on lib/main.dart file and you will see that it will open up in a separate tab.

Coding our app

Now let’s explore the code present in lib/main.dart file. Inside lib/main.dart file if you search for the main()function you will find the following code:

void main() {
    runApp(MyApp());
}

If you are familiar with dart, you might know that this is the main entry point of the program. This is the function that will be called first when starting the application. To learn more about dart, visit the Dart Tutorials page.

The runApp() function takes the widget (MyApp in this case) and renders it on screen.

In the following lines let’s explore the code inside MyApp class:

class MyApp extends StatelessWidget {
  const MyApp({Key? key}) : super(key: key);
  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      title: 'Flutter Demo',
      theme: ThemeData(
        primarySwatch: Colors.blue,
      ),
      home: const MyHomePage(title: 'Flutter Demo Home Page'),
    );
  }
}

The MyApp class extends StatelessWidget. We will not be explaining much about StatelessWidget in this tutorial. The next line is a simple constructor and the following lines contain some code that might interest us.

The build() function is a mandatory function that should be implemented when creating a new widget. It takes the context and widget that it will be rendering on screen. Here it simply returns an MaterialApp with title and home. MaterialApp widget is a built-in widget that is provided by the Flutter framework. It renders an empty flutter page with label text provided in the parameter: title.

Then it takes a theme and a home parameter. The theme parameter takes a ThemeData object (note theme is a parameter of the ThemeData object) with a color value given (In this case primarySwatch value is initialized to Colors.blue) that is then used to color the UI.

The final value home is the page that will be rendered on screen when you load the Home screen. The code that home is initialized with (MyHomePage(title: 'Flutter Demo Home Page')) creates a new instance of that class in the home, with the parameter title passed to the constructor of the MyHomePage class.

MyHomePage class is a stateful widget. We will not be going into details about how the class MyHomePage works, the stateful widget is basically used when an element of the UI needs to be updated. Some readers might find it a bit confusing. So, we will be replacing the class MyHomePage with another stateless widget which will display a button which will respond on click with the message “Hello Flutter”.

Replacing MyHomePage with MaterialButton

You can safely delete the class MyHomePageand _MyHomePageState classes from the lib/main.dartfile. Then reassign the value of home inside class MyApp to our button. The final code will look like this:

void main() {
  runApp(MyApp());
}

class MyApp extends StatelessWidget {
  const MyApp({Key? key}) : super(key: key);
  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      title: 'Flutter Demo',
      theme: ThemeData(
        primarySwatch: Colors.blue,
      ),
      home: Center(
        child: ElevatedButton(
          onPressed: () {
            showDialog(
                context: context,
                builder: (context) => const AlertDialog(
                      title: Text('Hello Flutter'),
                    ));
          },
          child: const Text('Hello World'),
        ),
      ),
    );
  }
}

Let’s take a look at the code we just added.

Center(
    child: ElevatedButton(
        onPressed: () {
        showDialog(
            context: context,
            builder: (context) => const AlertDialog(
                    title: Text('Hello Flutter'),
                ));
        },
        child: const Text('Hello World'),
    ),
)

The Center widget which is wrapping the ElevatedButton widget (which is a stateless widget) here helps us center the button in the middle of the screen. It is a simple but very useful widget provided by the Flutter framework.

Then the ElevatedButton widget takes two parameters: onPressed that is a callback function that is called when the button is clicked by the user, child: which is the text that will be displayed by the button. If you like to learn more about the ElevatedButton widget, you can visit the ElevatedButton Widget Article page.

The callback function takes a value as a parameter, using which it determines the state of the app. After the showDialog function is called, the app interracts with the user by displaying a simple alert dialog with the message “Hello Flutter”. The last parameter inside the constructor Text is also a stateless widget which is used to display text on the screen.

That’s it. Now that we’ve done with all the coding parts of this tutorial, we will now discuss how to run this app.

Testing the App

In this part of the tutorial we will go over how to test or run the app once it is created. Make sure you have an emulator or a device connected.

Using Terminal

Using the terminal run:

flutter run

If everything is setup properly you should see that the app is building and starting on the emulator or the connected device.

Using VS Code

  • Invoke Run > Start Debugging or press F5.

Then you would see the button displayed on screen with the label “Hello World.” On clicking the button its callback function get’s invoked and so the user gets a simple alert dialog with the label “Hello Flutter”.

Congratulations. You learned how to build a simple “Hello World” app with Flutter.

Summary

In this tutorial we went over the basics of building an Android application with flutter. As a next step you can take this app to next level by doing more complicated user interface and implementing a custom stateful widget.
